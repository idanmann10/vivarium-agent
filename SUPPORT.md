# Support for Vivarium Agent

Vivarium Agent is pre-1.0 and local-first. Use the public support paths below for reproducible local issues,
operator questions, and scoped feature requests. Do not share provider keys, credential values, private-world
content, internal URLs, or unredacted run evidence in public channels.

## Bugs

Open a bug report when a command, daemon route, runtime primitive, provider path, credential flow, or world
integration behaves incorrectly. Include exact commands, commit SHA, environment details, and redacted output.

Use the bug report template:

https://github.com/idanmann10/vivarium-agent/issues/new?template=bug_report.yml

## Feature Requests

Open a feature request for scoped CLI, daemon, runtime, provider, credential, world, or documentation work. Describe
the smallest useful behavior change and the validation that would prove it works.

Use the feature request template:

https://github.com/idanmann10/vivarium-agent/issues/new?template=feature_request.yml

## Discussions

Use GitHub Discussions for operator questions, design proposals, RFCs, and live-readiness handoff discussion.
Discussions are the right place for questions that are not yet reproducible bugs or implementation-ready features.

https://github.com/idanmann10/vivarium-agent/discussions

## Security

Report vulnerabilities, credential handling bugs, prompt-injection bypasses, and accidental secret exposure through
the private security policy route, not public issues or Discussions.

Read [SECURITY.md](SECURITY.md) before reporting:

https://github.com/idanmann10/vivarium-agent/security/policy

## Live Readiness

For provider smokes, internal credential smokes, world publication evidence, or v1 completion blockers, start with
the live-readiness guide and include the redacted `doctor --live` output when asking for support.

Read [docs/guides/live-readiness.md](docs/guides/live-readiness.md).
