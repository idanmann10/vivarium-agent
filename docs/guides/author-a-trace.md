# Author a Trace

Traces annotate steps, observations, reasons, pitfalls, and alternatives.

## File

World traces live as `TRACE.md` files. They are deliberate teaching artifacts,
not raw transcripts. Include frontmatter for id, title, domain, visibility, and
contributor.

## Content

Document the workflow in a way another agent can reuse:

- `prerequisites`: what must be true before starting.
- `TraceStep`: one action plus one annotation explaining why it matters.
- `pitfalls`: mistakes or misleading signals to watch for.
- `alternatives`: valid branches when context differs.

## Evidence

Prefer traces extracted from successful runs with clear observations and
validation. If a trace is generated by Dream, review the annotations before
publishing so it teaches reasoning instead of merely replaying commands.
