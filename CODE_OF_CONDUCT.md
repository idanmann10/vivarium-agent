# Code of Conduct

## Scope

The Vivarium Agent community should stay focused on rigorous, respectful collaboration around local-first agent
runtime work. This Code of Conduct applies to issues, pull requests, Discussions, reviews, docs, examples, and
community support spaces.

## Expected Behavior

- Be direct, technical, and respectful.
- Critique code, evidence, and reasoning rather than people.
- Keep security reports private when disclosure could harm users.
- Preserve privacy when sharing logs, traces, runs, or live-readiness evidence.
- Accept maintainer decisions about safety, release readiness, and contribution scope.

## Unacceptable Behavior

- harassment, threats, personal attacks, or discriminatory language.
- Publishing secrets, credentials, private data, or exploit details in public threads.
- Misrepresenting local tests as live v1 completion evidence.
- Repeatedly derailing issues or reviews after maintainers have set scope.

## Enforcement

Maintainers may edit, hide, or remove content; close issues or pull requests; block accounts; or report abuse to
platform moderators. If you need to report a conduct issue, contact a maintainer privately with links, dates, and
the smallest amount of sensitive context needed to evaluate the issue.
